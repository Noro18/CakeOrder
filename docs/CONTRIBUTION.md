# Contributing to OrderManagementCake

*(Tetum iha kraik)*

Thank you for your interest in contributing! This guide will help you get started with the contribution process, even if you are a beginner.

## Contribution Workflow

To keep the project organized and professional, please follow these five steps:

### 1. Clone the Repository
Start by cloning the project to your local machine:
```bash
git clone <repository-url>
cd OrderManagementCake
```

### 2. Create a Branch
Before making any changes, create a new branch. We use a professional naming convention with prefixes to categorize the work.

**Branch Naming Convention:** `prefix/short-description`

| Prefix | Use Case | Example |
| :--- | :--- | :--- |
| `feature/` | Developing a new feature | `feature/login` |
| `bugfix/` | Fixing a bug in the existing code | `bugfix/crash-on-back` |
| `hotfix/` | Urgent fix for a critical issue | `hotfix/payment-error` |
| `docs/` | Documentation changes only | `docs/contribution-guide` |
| `refactor/` | Code changes that improve structure without changing behavior | `refactor/ui-components` |
| `chore/` | Maintenance tasks (e.g., updating dependencies) | `chore/update-gradle` |
| `test/` | Adding or updating tests | `test/unit-tests-main` |

### 3. Commit Your Changes
Make sure to commit after each significant change. We use the following format for clear and professional commit messages:

**Format:** `type(scope): description`

| Type | Description |
| :--- | :--- |
| `feat` | A new feature |
| `fix` | A bug fix |
| `docs` | Documentation changes |
| `style` | Formatting, missing semi colons, etc; no code change |
| `refactor` | Refactoring production code |
| `test` | Adding missing tests, refactoring tests |
| `chore` | Updating build tasks, package manager configs, etc |

**Example:**
`feat(auth): implement user login validation`

### 4. Push and Make a PR
Once your changes are ready, push your branch and open a Pull Request (PR) on GitHub:
```bash
git push origin your-branch-name
```
Go to the repository on GitHub to finalize the PR creation.

### 5. Review and Merge
The project owner will review your Pull Request. If everything looks good and follows the standards, it will be merged into the `main` branch!

# Kontribui ba OrderManagementCake

Obrigadu barak ba Ita-boot nia interese atu kontribui! Gia ida-ne'e sei ajuda Ita-boot atu hahú prosesu kontribuisaun, maske Ita-boot sei foin-sa'e (beginner).

## Fluksu Servisu Kontribuisaun

Atu mantein projetu ne'e organizadu no profisionál, favór tuir hakat lima tuirmai ne'e:

### 1. Clone Repozitóriu
Hahú hodi clone projetu ne'e ba Ita-boot nia komputador local:
```bash
git clone <repository-url>
cd OrderManagementCake
```

### 2. Kria Branch Foun
Antes halo mudansa ruma, kria branch foun ida. Ita uza konvensaun naran profisionál ho prefiksu atu kategoria servisu ne'e.

**Konvensaun Naran Branch:** `prefiksu/deskrisaun-badak`

| Prefiksu | Kazu Uza | Ezemplu |
| :--- | :--- | :--- |
| `feature/` | Dezenvolve funsaun (feature) foun | `feature/login` |
| `bugfix/` | Hadia bug iha kódigu ne'ebé iha ona | `bugfix/crash-on-back` |
| `hotfix/` | Hadia urjente ba problema krítiku | `hotfix/payment-error` |
| `docs/` | Mudansa dokumentasaun de'it | `docs/contribution-guide` |
| `refactor/` | Mudansa kódigu ne'ebé hadia estrutura maibé la muda nia hahalok | `refactor/ui-components` |
| `chore/` | Servisu manutensaun (ezemplu: atualiza dependénsia sira) | `chore/update-gradle` |
| `test/` | Aumenta ka atualiza teste sira | `test/unit-tests-main` |

### 3. Commit Ita-boot nia Mudansa
Hala'o commit depois de mudansa signifikativu ida-idak. Ita uza formatu tuirmai ne'e atu mensajen commit sai klaru no profisionál:

**Formatu:** `tipu(skopu): deskrisaun`

| Tipu | Deskrisaun |
| :--- | :--- |
| `feat` | Funsaun foun |
| `fix` | Hadia bug |
| `docs` | Mudansa dokumentasaun |
| `style` | Formatasaun, falta vírgula (semi-colon), nst; la'ós mudansa kódigu |
| `refactor` | Refactor kódigu produsaun |
| `test` | Aumenta teste ne'ebé falta, refactor teste sira |
| `chore` | Atualiza servisu build, konfigurasaun package manager, nst |

**Ezemplu:**
`feat(auth): implementa validasaun login uza-na'in`

### 4. Push no Kria PR
Wainhira mudansa sira prontu ona, push Ita-boot nia branch no loke Pull Request (PR) iha GitHub:
```bash
git push origin naran-ita-nia-branch
```
Bá repozitóriu iha GitHub atu finaliza kriasun PR.

### 5. Revisaun no Merge
Na'in ba projetu sei revisa Ita-boot nia Pull Request. Se buat hotu di'ak no tuir padraun sira, sei merge ba branch main!

Boa kontribuisaun! 🍰
